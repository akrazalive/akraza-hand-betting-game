рҹ§Ә What Questions Will Be Asked in Assessment?
Based on "scalability" and "feature-ready" requirements, expect questions like:

Easy Questions (Understanding your code):
"How would you add a new tile type (e.g., Seasons tiles)?"

Answer: Modify tileDeck.ts вҶ’ createFullDeck(), update types.ts

"How would you change non-number tiles to start at value 3 instead of 5?"

Answer: Change in tileDeck.ts вҶ’ initializeNonNumberTileValues()

"Where would you add a new game over condition (e.g., time limit)?"

Answer: Add to shouldEndGame() in scoring.ts

Medium Questions (Adding features):
"Add a 'Double or Nothing' feature after winning"

Need to modify: GameBoard.tsx (add button), useGame.ts (new function), scoring.ts (new calculation)

"Add sound effects for reshuffling"

Answer: Add soundService.playReshuffle() in gameEngine.ts вҶ’ reshuffle() function

"Add a betting streak counter (consecutive wins)"

Answer: Add streak to GameState in types.ts, update in useGame.ts when resolving

Hard Questions (Complex changes):
"Add a 'Multiplayer' mode where two players bet against each other"

Would need: New MultiplayerGameState, WebSockets, separate player components

"Add an AI opponent that learns betting patterns"

Would need: AI logic in new file, integrate with betting system

"Implement undo/redo for betting moves"

Would need: History stack in useGame.ts, save game states

Database/Leaderboard Questions:
"How would you add filtering by date range?"

Answer: Modify app/api/scores/route.ts to accept date parameters

"How would you add pagination for more than 5 scores?"

Answer: Change Leaderboard.tsx to fetch page by page
